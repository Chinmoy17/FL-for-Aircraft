# Frontend (reserved)

This folder is reserved for a React dashboard that visualises every phase of the
**Federated Learning for Aircraft Engine PHM** project. **No code lives here
yet** — only this contract document. The frontend will be scaffolded once the
baseline (P3–P7) is complete and the metric schema has stabilised.

## What the frontend will consume

The frontend reads **JSON only**. It never imports Python, never re-runs models,
and never parses Markdown. The contract is:

| Path | Purpose | Stability |
| --- | --- | --- |
| `../results/summary.json` | Top-level aggregate. The frontend's main entry point. | Regenerated by `scripts/build_results_summary.py`. |
| `../results/NN_<phase>/metrics.json` | One per phase (`00_eda`, `01_data`, `02_smoke`, `03_centralized`, `04_local_only`, `05_fedavg`, `06_non_iid`, `rq2_*`, …). | Written by each phase's script. |
| `../results/NN_<phase>/*.png` | Plot images referenced by `metrics.json` under `artifacts`. | Static — load with `<img src="...">`. |
| `../results/NN_<phase>/*.csv` | Flat metric tables for human inspection. The frontend does **not** need to parse these. | — |

## JSON schema

Per-phase `metrics.json` (and the entries under `summary.json["phases"]`) follow
the dataclass defined in
[`../src/fl_aircraft/utils/results.py`](../src/fl_aircraft/utils/results.py)
(`PhaseMetrics`). All fields except `phase_id` and `phase_name` are optional —
each phase populates whatever applies:

```jsonc
{
  "phase_id":       "02_smoke",
  "phase_name":     "Phase 2 — Centralized smoke run",
  "generated_at":   "2026-06-18T12:34:56+00:00",
  "interpretation": "Tooltip-friendly one-paragraph summary.",
  "artifacts":      { "loss_curve_png": "results/02_smoke/loss_curve_fd001.png" },
  "subset":         "FD001",
  "config":         { "epochs": 1, "batch_size": 256, "lr": 0.001, "seed": 42 },
  "timing":         { "train_seconds": 1.5, "train_seconds_per_epoch": 1.5 },
  "summary":        { "model_name": "MultiTaskCNN", "model_n_parameters": 30018 },
  "train":          { "epochs": [{ "epoch": 1, "loss_total": 640.77 }] },
  "test": {
    "rul":   { "rmse": 62.7, "mae": 52.7, "nasa_score": 45300.4 },
    "fault": { "auprc": 0.845, "f1": 0.4, "precision": 0.25, "recall": 1.0 }
  },
  "per_subset": { "FD001": { /* … */ } },
  "per_client": { "client_1": { /* … */ } }
}
```

`summary.json` adds a wrapper:

```jsonc
{
  "project":      "Federated Learning for Aircraft Engine PHM",
  "generated_at": "...",
  "git_commit":   "abc1234",
  "phases":       { "00_eda": { /* … */ }, "01_data": { /* … */ }, ... }
}
```

## Suggested initial pages

1. **Overview** — project blurb + list of completed phases (from `phases` keys),
   each linking to its detail page.
2. **EDA** (`00_eda`) — per-subset stats table + the 6 PNG figures (lifetime,
   correlation, regimes, trajectories, RUL, fault imbalance).
3. **Data pipeline** (`01_data`) — per-client engine count / window count /
   fault rate, with the imbalance figure as the headline visual.
4. **Smoke run** (`02_smoke`) — config card + loss-curve image + test-metric
   tiles + interpretation as the page footer.
5. **Centralized / local-only / FedAvg** (`03_*` / `04_*` / `05_*`) —
   side-by-side comparison cards.
6. **RQ experiments** (`rq2_*`, `rq5_*`, `rq3_*`) — before/after diff tables.

## Suggested stack (non-binding)

- Vite + React 19 + TypeScript.
- Tailwind + shadcn/ui for the layout.
- Recharts or Visx for the loss / metric curves.
- No backend — read JSON via `fetch()` from a sibling `results/` folder served
  statically (the Azure free-tier Docker image will serve both the static
  frontend and the `results/` tree from the same nginx container).

## When you do build it

```powershell
cd frontend
npm create vite@latest . -- --template react-ts
npm install
# ... build out pages reading ../results/summary.json
```

Keep this folder tree small (no committed `node_modules`); the lockfile
(`package-lock.json`) plus `.gitignore` of `node_modules/` and `dist/` is
all the configuration needed.
